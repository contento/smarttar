We do NOT include pre-built libraries for older compilers.

Here is an example of building a DLL library with Visual 4.2.

Make sure your PATH includes MSDEV\BIN.  The INCLUDE and LIB environment
variables should be something like this:

INCLUDE=C:\MSDEV\INCLUDE;C:\MSDEV\MFC\INCLUDE
LIB=C:\MSDEV\LIB;C:\MSDEV\MFC\LIB

Go to the V42 directory.  Execute BUILDV42 with no parameters to display
a help screen showing available libraries you can build.  To build the
MFC DLL:

buildv42 21MFC

To build the debug version of the DLL:

buildv42 21MFCD

BUILDV42 defaults to building libraries.  The -lib argument is optional.

Please refer to your compiler's documention for using other compilers.

Borland C++ Builders NOTE:

Here is my BCC32.CFG and TLINK32.CFG located in my CBUILDER\BIN directory:

BCC32.CFG
-Ic:\cbuilder\include;c:\cbuilder\include\vcl;c:\al212\h
-Lc:\cbuilder\lib;c:\al212\bcb10

TLINK32.CFG
-Lc:\cbuilder\lib;c:\cbuilder\lib\obj

When building libraries, make sure you only have ONE compiler in your PATH.
I've goofed by having both Borland C++ 5.0 and BCB in my path.  Not good.
