OVERVIEW
--------

ArchiveLib 2.11 supports two methods of encryption.  Zip compatible
encryption uses the same algorithm used by PKZip to encrypt files.  You
should always use this form of encryption when working with ZIP files.

Xor encryption is performs a very simple XOR of the contents of an archive
against a key of your choice.  This encryption is fairly simple to break,
particularly if the key is short.

At this time, we cannot export libraries using Zip compatible encryption
outside of the US.  Likewise, you cannot export products that support this
form of encryption.  We're working on this problem.

The simple interface has no provision for encryption at this time.


USAGE
-----

Using encryption is very easy, and can be added to your application with
little or no effort.

To encrypt data when creating an archive, you simply have to ensure that
the ALEntry * for each file has a pointer to a crypto engine.  There are
two ways to do this.  The first is to add a crypto engine to the ALEntryList
toolkit when the list is created.  Files added to the entry list from that
point on will have a crypto engine, and will be encrypted automatically when
compressed.

The second way to do this is to create crypto engines by hand, and place
them into an ALEntry * by hand, in the mpCrypto pointer member.  This
crypto engine will be destroyed by the library when the ALEntry * is
destroyed, so be sure it isn't an automatic object, and be sure you
don't attempt to destroy it.
